The client operates power distribution infrastructure across several European countries and needed a platform to manage the IoT devices deployed across their grid: sensors, remote terminal units, and intelligent electronic devices reporting telemetry at variable frequencies. The existing monitoring was manual, fragmented, and unable to scale to the device count they were rolling out.

I designed the backend architecture as a Kafka-backed event pipeline, with Java Spring Boot microservices handling ingestion, normalization, and alert evaluation. The key constraint was latency: operators needed alert notifications within one second of a device reading crossing a threshold. TimescaleDB handled the time-series telemetry storage, with a separate PostgreSQL database for device registry and configuration. A React frontend provided the operator dashboard.

The platform launched on schedule with 99.95% ingestion SLA across the first three months. *The architectural decision I'm most satisfied with*: modeling device state as a stream of immutable events rather than a mutable record, which made the alert rule engine straightforward to build and the historical replay of device behaviour easy to implement for debugging.
